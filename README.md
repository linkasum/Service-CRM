# Система учёта для сервисных центров

CRM для управления сервисным центром по ремонту электроники.

## Возможности

### Касса и смены
- Открытие/закрытие кассовых смен
- Учёт наличных и безналичных платежей
- Приход, расход, переводы
- Цепочка смен с переходящим остатком
- Месячная сводка по кассе

### Заказы
- Создание, редактирование, статусная модель (Новый → В работе → Готов → Выдан)
- Назначение мастера и приёмщика
- Услуги и запчасти
- Клиентская база (имя, телефон, email)
- Печать документов: акт выполненных работ, квитанция, счёт, акт диагностики
- Системные комментарии и история

### Зарплата
- Формула: `({cash_net} + {card_net}) × 0.4` — 40% от чистого дохода мастера
- Поддержка переменных: `{cash_net}`, `{card_net}`, `{payments_net}`, `{total_cost}`, `{parts_cost}`
- Фиксированная ставка за смену (4 000 ₽)
- Автоначисление при выдаче заказа
- Два периода выплат: 1–15 число и 16–конец месяца
- Детализация по каждому заказу (начислено / удержано)
- Учёт удержаний за запчасти

### Склад запчастей
- Приход, расход, списание в заказ
- Списание запчасти автоматически вычитает 40% от стоимости из ЗП мастера
- Учёт остатков

### Telegram бот
- Уведомления о новых заказах и смене статусов
- Настройка через `TELEGRAM_BOT_TOKEN` в `.env`
- Привязка Telegram Chat ID к сотруднику в профиле

### График работы
- Посменный график (3/3)
- Назначение сотрудников по дням

### Печатные формы (HTML → PDF)
- Акт выполненных работ (`work_act`)
- Квитанция (`receipt`)
- Счёт на оплату (`invoice`) с QR-кодом СБП
- Акт диагностики (`diagnostic_act`)
- Шаблоны редактируются через визуальный редактор

### Аналитика
- Дашборд: выручка, заказы, загрузка мастеров
- Отчёт по зарплате мастеров
- История кассовых смен

## Установка

### Требования
- Docker и Docker Compose
- PostgreSQL (или использовать контейнер)

### Быстрый старт

```bash
# Клонировать репозиторий
git clone <repo-url>
cd qwencrm

# Создать пустую базу
psql -U postgres -c "CREATE DATABASE qwencrm"

# Запустить
docker compose -f docker-compose.dev.yml --profile bot up -d

# Применить миграции
docker compose -f docker-compose.dev.yml --profile bot exec backend alembic upgrade head
```

### Инициализация пустой базы

```bash
psql -U postgres -d qwencrm -f init_db.sql
```

Либо запустить скрипт `./init_empty_db.sh` для автоматического создания.

## Переменные формул зарплаты

| Переменная | Описание |
|-----------|---------|
| `{cash_net}` | Чистый нал (доход − расход) |
| `{card_net}` | Чистый безнал |
| `{payments_net}` | Общий чистый доход |
| `{total_cost}` | Полная стоимость заказа |
| `{parts_cost}` | Стоимость запчастей |
| `{orders_count}` | Количество заказов |
| `{base_salary}` | Базовая ставка |

## Переменные печатных форм

| Переменная | Описание |
|-----------|---------|
| `{order_id}` | Номер заказа |
| `{client_name}` | Имя клиента |
| `{client_phone}` | Телефон |
| `{device_brand}` | Бренд устройства |
| `{device_model}` | Модель |
| `{total_cost}` | Стоимость заказа |
| `{complaint}` | Неисправность |
| `{order_date}` | Дата заказа |
| `{items_table}` | Таблица услуг и запчастей |
| `{warranty_days}` | Гарантия (дней) |

## Бэкап

```bash
# Полный бэкап (файлы + БД)
bash backup_full.sh

# Только БД
bash backup_db.sh
```

Бэкапы сохраняются на NAS через `smbclient`.

## Импорт из Excel

Для массового импорта заказов используется локальный файл `june.xlsx`:

1. Файл размещается в `/opt/qwencrm/backend/june.xlsx`
2. Запускается скрипт импорта через `docker exec`
3. Все транзакции создаются в `cash_transactions`
4. ЗП пересчитывается по формуле
