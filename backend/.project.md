## О проекте

CRM-система для сервисного центра по ремонту техники. Управление заказами, складом, кассой, зарплатами, клиентами, отчётами.

### Стек технологий
- **Backend:** FastAPI + SQLModel + PostgreSQL (Python 3.11)
- **Frontend:** React + Vite + TypeScript + Ant Design + Recharts
- **Печать PDF:** ReportLab (DejaVuSans для кириллицы)
- **Редактор шаблонов:** ReactQuill (WYSIWYG)
- **Сборка:** Vite (SPA с проксированием API через `run_spa_server.py`)
- **Тесты:** Playwright (Chromium)

**Как работает автоначисление:**
1. При смене статуса на "issued" через кассу вызывается `/api/salary/assignment/auto-assign/{order_id}`
2. Сначала ищет личную формулу сотрудника (`user.salary_config_id`)
3. Если нет — берёт активную формулу типа formula
4. Рассчитывает зарплату и создаёт запись SalaryRecord

## Важно знать

1. **НЕ использовать `from __future__ import annotations`** — ломает SQLAlchemy
2. **Кириллический шрифт:** `backend/fonts/DejaVuSans.ttf`
3. **PDF endpoint `from-template`** должен возвращать `FileResponse`, не JSON
4. **При добавлении запчасти/услуги** — пересчитывать `total_cost` заказа
5. **Смена в кассе** — начальный баланс = остаток предыдущей закрытой смены
6. **SPA сервер** проксирует `/api/*` на `http://127.0.0.1:8000`
7. **Бинарные файлы (PDF, Excel)** — передавать с `Content-Length` через прокси
8. **Токен** хранится в `localStorage` как `token`
9. **Зарплаты** — теперь неограниченное количество активных формул, индивидуальные для каждого сотрудника
10. **Настройки** — боковое меню вместо табов (15 разделов)
